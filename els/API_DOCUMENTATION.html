📖 API Overview

The Electronic Banking System (ELS) API is a comprehensive RESTful web service that manages all banking operations. The API follows RESTful conventions and returns JSON responses.

✓ Fully Documented: All endpoints are documented with Swagger/OpenAPI 3.0 specification, including data models, request/response schemas, and example values.

🗂️ Data Models

User Model

Represents a user account in the ELS system
  • id - Long - Unique identifier (Primary Key)
  • email - String - Email address (Unique constraint)
  • first_name - String - User's first name
  • last_name - String - User's last name
  • phone_number - Integer - Contact phone number
  • bsn - Integer - Dutch BSN (Unique constraint)
  • role - Integer - User role (0=customer, 1=employee, 2=admin)
  • approved - Boolean - Account approval status
  • created_at - LocalDateTime - Record creation timestamp

Account Model

Represents a bank account
  • id - Long - Unique identifier (Primary Key)
  • customer_id - Long - Owner of the account (Foreign Key to User)
  • iban - String - International Bank Account Number (Unique constraint)
  • account_type - String - Type of account (SAVINGS, CHECKING, etc.)
  • balance - BigDecimal - Current account balance
  • absolute_transfer_limit - BigDecimal - Maximum per transaction
  • daily_transfer_limit - BigDecimal - Maximum per day
  • active - Boolean - Account status
  • created_at - LocalDateTime - Record creation timestamp

Transaction Model

Represents a financial transaction
  • id - Long - Unique identifier (Primary Key)
  • from_account_id - Long - Source account (Foreign Key)
  • to_account_id - Long - Destination account (Foreign Key)
  • initiated_by_user_id - Long - User who initiated (Foreign Key)
  • amount - BigDecimal - Transaction amount
  • transaction_type - String - Type (TRANSFER, DEPOSIT, WITHDRAWAL, etc.)
  • currency - String - Currency code (EUR, USD, etc.)
  • timestamp - LocalDateTime - Transaction time
  • status - String - Status (PENDING, COMPLETED, FAILED, REJECTED)
  • description - String - Transaction notes

ATM Session Model

Represents an ATM login session
  • id - Long - Unique identifier (Primary Key)
  • customer_id - Long - User who logged in (Foreign Key to User)
  • login_time - LocalDateTime - Session login timestamp
  • logout_time - LocalDateTime - Session logout timestamp
  • active - Boolean - Current session status

🔗 Entity Relationships

User (1) ──owns──> (N) Account
User (1) ──initiates──> (N) Transaction
User (1) ──uses──> (N) ATM_Session
Account (1) ──sends──> (N) Transaction
Account (1) ──receives──> (N) Transaction

🔐 Security

⚠️ Important: This application has Spring Security enabled. Make sure to configure authentication properly in production environments.

Security Features

  • Spring Security framework for authentication and authorization
  • Support for role-based access control (RBAC)
  • Password hashing for user accounts
  • BSN (Burgerservicenummer) unique constraint for Dutch identification
  • Email and IBAN unique constraints

Production Recommendations

  • Always use HTTPS in production
  • Implement rate limiting for API endpoints
  • Use strong password policies
  • Enable audit logging for sensitive operations
  • Implement request signing for critical endpoints
  • Use API keys or OAuth2 for client authentication
